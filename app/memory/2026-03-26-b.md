### 5. FinIQ Agent Completely Rebuilt with LLM ‚úÖ
**File**: `server/agents/finiq-agent.mjs`

**Architecture**:
```
User Query ‚Üí LLM Analysis (intent + SQL) ‚Üí Execute SQL ‚Üí LLM Summarization ‚Üí Response
```

**Key Functions**:

1. **`processQuery(query, ctx)`**
   - Main entry point
   - Checks if LLM is available (Anthropic API key)
   - If yes: Uses `analyzeQueryWithLLM()` ‚Üí execute ‚Üí `summarizeResultsWithLLM()`
   - If no: Falls back to `fallbackProcessQuery()` (keyword-based)

2. **`analyzeQueryWithLLM(query)`**
   - Sends full schema context + user query to Claude
   - Asks LLM to return JSON: `{ intent, entity, period, sql, explanation }`
   - Uses `claude-sonnet-4-20250514` model
   - Handles JSON extraction from markdown code blocks
   - Returns structured analysis

3. **`summarizeResultsWithLLM(query, analysisResult, data)`**
   - Sends query + first 10 data rows to Claude
   - Asks for concise 1-2 sentence executive summary
   - Focuses on key metrics, trends, insights
   - Uses Mars-appropriate language (no "replace", "fragmented")
   - Returns plain text summary

4. **`calculateKPIs(intent, data)`**
   - Computes KPI metrics from query results
   - For PES/NCFO/product: YTD/Periodic growth %
   - For variance: Actual vs Replan difference
   - Returns array of KPI objects with `trend: 'up'|'down'`

5. **`fallbackProcessQuery(query)`**
   - Simple keyword-based intent classification
   - Regex entity/period extraction
   - Direct database calls (getPLData, getNCFOData, etc.)
   - Used when `ANTHROPIC_API_KEY` not set

**Response Format**:
```json
{
  "success": true,
  "query": "...",
  "intent": "pes|ncfo|variance|product|adhoc",
  "entity": "Mars Inc",
  "period": "P06",
  "summary": "LLM-generated executive summary",
  "kpis": [
    { "label": "YTD CY", "value": 123.45, "format": "currency", "trend": "up" }
  ],
  "data": [...],
  "dataSource": "SQL query",
  "rowCount": 27,
  "timestamp": "...",
  "llmUsed": true
}
```
